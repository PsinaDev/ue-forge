# Include Optimizer

**[English version](README.md)** · [← Назад к UE Forge](../../../README_RU.md)

Оптимизация директив `#include` в C++ исходниках проекта Unreal Engine.

## Возможности

- **`UE_INLINE_GENERATED_CPP_BY_NAME`** — добавляет отсутствующие inline-макросы в `.cpp`, чей парный `.h` содержит `GENERATED_BODY()`. Проверка версии движка (требуется UE 5.1+). Точка вставки учитывает препроцессор — всегда на верхнем уровне, никогда внутри `#if` блоков
- **Замена `CoreMinimal.h`** — анализирует реальное использование типов (100+ паттернов: контейнеры, математика, делегаты, UObject-макросы, умные указатели, типы движка) и заменяет catch-all `CoreMinimal.h` только на конкретные нужные заголовки, плюс `CoreTypes.h` как базовый
- **Удаление дубликатов `#include`** — находит и удаляет повторяющиеся инклюды в пределах одного файла
- **Исправление препроцессорных блоков** — обнаруживает `UE_INLINE_GENERATED_CPP_BY_NAME` внутри `#if` / `#ifdef` / `#ifndef` блоков и выносит их за пределы `#endif`
- **Сканирование плагинов** — при выборе `.uproject` опционально сканирует `Plugins/` рекурсивно. Плагины группируются по категориям с индивидуальными чекбоксами для исключения
- **Diff-превью** — категоризированные изменения со сворачиваемыми группами. При включении плагинов изменения группируются по источнику (Проект / каждый плагин), затем по категории оптимизации
- **Бэкап** — `Source/` → `Source_backup/` перед модификацией, включая отдельные бэкапы для каждого плагина

## Запуск

```bash
python -m ue_forge.include_optimizer
# или в хосте:
python -m ue_forge   # → "Инклюды" в сайдбаре
```

1. Выберите `.uproject`, `.uplugin` или директорию `Source/`
2. Включите/выключите категории оптимизации
3. (Для `.uproject`) Включите «Плагины» и снимите галки с ненужных
4. Просмотрите изменения в diff-превью
5. Нажмите **Оптимизировать**

## Логика замены CoreMinimal.h

Оптимизатор удаляет комментарии и строки `#include` из содержимого файла, затем сканирует 100+ regex-паттернов, сопоставленных конкретным заголовкам:

| Паттерн использования | Заголовок |
|---|---|
| `TArray<>` | `Containers/Array.h` |
| `FString` | `Containers/UnrealString.h` |
| `FVector` | `Math/Vector.h` |
| `UE_LOG` | `Logging/LogMacros.h` |
| `TSharedPtr<>` | `Templates/SharedPointer.h` |
| `UCLASS()` | `UObject/ObjectMacros.h` |
| `AActor` | `GameFramework/Actor.h` |
| ... | (100+ маппингов всего) |

`CoreTypes.h` всегда включается как безопасная база (`int32`, `uint8`, `TEXT()`, `TCHAR`, `FORCEINLINE`, `check`/`ensure`, макросы платформ).

Заголовки, уже явно присутствующие в файле, не дублируются.

## Осведомлённость о препроцессоре

Оптимизатор отслеживает глубину вложенности `#if` / `#ifdef` / `#ifndef` / `#endif` для каждой строки. Это предотвращает два класса багов:

1. **Вставка** — `UE_INLINE_GENERATED_CPP_BY_NAME` вставляется после последнего `#include` на глубине 0 (верхний уровень), а не после `#include` внутри `#if WITH_EDITOR`
2. **Обнаружение** — категория «Preprocessor Block Fix» находит существующие `UE_INLINE_GENERATED_CPP_BY_NAME`, оказавшиеся внутри условных блоков, и предлагает перенести их за `#endif`
